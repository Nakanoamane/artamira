# プロンプト倉庫

## 設計

```
## 1. 要件定義

依頼内容をもとに要件をまとめてください。

## 2. 調査

- 現在のプロジェクトの実装内容を確認してください。
- 機能の実装に新しく必要な技術があれば、web検索をして適切な技術の選定を行なってください。
  - 技術に複数の候補があるときは、メリット・デメリットを洗い出して最も適している技術を採用する


## 3. 設計書

要件・現在の実装、技術選定の結果をもとに設計書を作成してください（docs/design/以下に.md形式）

### 設計書の内容（例）
- 要件
- 機能一覧
- DB設計
- バックエンド設計
- フロントエンド設計

### 設計書レビュー

作成した設計書と要件をレビューして修正してください

## 4. 実行計画

設計書をもとに実装計画を立て、設計書に追記してください
- 実装作業は最小単位に分割する
- 実装・テストを細かいサイクルで回す
- 現在の実装を確認してデグレーションが起こらないようにする
- テストの実行方法はREADME.mdを確認する

## 5. プロンプト作成

実装計画に沿ってAIに実装を依頼するためのプロンプトを作成し、設計書に追記してください

```

## 実装



##　テスト

```
テストの実行方法はREADME.mdを参照してください
```

```
設計書の内容と、現在のプロジェクトの実装を比較して、テストケースの追加・修正をしてください。

- 設計書の内容をもとに必要なテストケースを洗い出してください
- テストの実装を確認し、テストケースに過不足がないか確認してください
- 実装と比較しながらテストケースの追加・修正を行なってください
```


```
テストの結果をもっと詳しく理解するために、デバッグ用の記述を追加して確認したいです。

- 現在のテストの結果をまとめてください。
- どこまでができていて、どこからできていないのかを明確にしてください
- 設計書があれば、テストケースと設計書の記述が一致しているか確認してください
- 実装とテストの記述が一致しているのか確認してください
- テストの結果についてわからないことがあれば検索してください
- 適切な場所にデバッグ用の喜寿致死、原因を特定してください
```

## リファクタ

```
〇〇ファイルについて、レビューをしてください。
- このファイルはどのような役割がありますか
- このファイルの役割に対して、実装されている機能の量は適切ですか
- 一般的なreactの設計と比べたとき、このファイルに改善点はありますか
- 保守性・効率性を考えたとき、改善点はありますか
```
